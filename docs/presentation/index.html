Architecture Overview

"The overriding rule that makes this architecture work is The Dependency Rule."
â€” Robert C. Martin (Uncle Bob), Clean Architecture

We implemented a strict Onion Architecture to decouple logic from rendering.

  • Core (Model): Pure C++, Zero Dependencies.
  • Engine (View): RAII Wrappers for SDL2.
  • Game (Controller): State Machine & Glue.

Core Module (The Model)

Value Semantics

Optimized for Cache Locality. No dynamic allocation per tile.


// src/core/Tile.hpp
class Tile {
public:
    // Lightweight Value Object
    explicit Tile(int val) : value(val), merged(false) {}

    [[nodiscard]] int getValue() const { return value; }
    [[nodiscard]] bool isEmpty() const { return value == 0; }

private:
    int value;
    bool merged;
};

Engine Module (The View)

RAII Resource Management

Automatic cleanup of SDL resources using C++ destructors.


// src/engine/Context.hpp
class Context {
public:
    Context() {
        if (SDL_Init(SDL_INIT_VIDEO) < 0) throw std::runtime_error(...);
        TTF_Init();
    }

    ~Context() {
        // Automatically cleaned up on scope exit
        TTF_Quit();
        SDL_Quit();
    }
};
